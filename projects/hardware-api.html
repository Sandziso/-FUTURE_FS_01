Project Overview

Hardware Shop RESTful API is a comprehensive backend service designed for hardware store inventory management. Built with Java and JAX-RS, this API provides secure endpoints for product management, user authentication, and inventory tracking with a focus on performance and scalability.

👨‍💻 My Role: Sole Developer

I was the sole developer responsible for the entire project lifecycle:

  • API Architecture: Designed RESTful endpoints and data models
  • Database Design: Created normalized MySQL schema with relationships
  • Backend Development: Implemented Java services with JAX-RS
  • Authentication System: Built JWT-based secure authentication
  • Testing & Documentation: Comprehensive unit testing and API documentation
  • Deployment: Configured Maven build and Tomcat deployment

Technical Implementation

Database Architecture

Designed and implemented a comprehensive MySQL database with normalized tables:

  • Users: User management with roles (admin, staff, customer) and authentication
  • Products: Product catalog with categories, suppliers, and pricing
  • Categories: Product categorization with hierarchical support
  • Suppliers: Supplier information and contact details
  • Inventory: Stock levels, transactions, and inventory history
  • Orders: Order management with line items and status tracking

Security Implementation

  • JWT-based authentication with token expiration and refresh mechanisms
  • Role-based access control for different user types
  • Password hashing with bcrypt algorithm
  • Input validation and sanitization for all endpoints
  • SQL injection prevention using prepared statements
  • CORS configuration for secure cross-origin requests

System Architecture

Layered Architecture

The API follows a clean layered architecture with separation of concerns:

Application Layers:

┌─────────────────┐
│   REST Layer    │  ← JAX-RS Resources, Request/Response handling
├─────────────────┤
│  Service Layer  │  ← Business logic, validation, transaction management
├─────────────────┤
│   DAO Layer     │  ← Data access objects, database operations
├─────────────────┤
│ Database Layer  │  ← MySQL database, connection pooling
└─────────────────┘

Request Flow

Standardized request processing pipeline:

  • Authentication Filter: JWT validation and user context setup
  • Request Processing: Input validation and parameter parsing
  • Business Logic: Service layer processing with transaction management
  • Data Access: Database operations through DAO layer
  • Response Building: Standardized JSON response formatting
