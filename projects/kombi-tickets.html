Project Overview

Kombi Tickets is a modern web platform designed to simplify kombi (minibus) travel across Eswatini. It combines an interactive map, real‑time seat selection, QR code tickets, and a driver scanner to create a seamless experience for both passengers and drivers.

👨‍💻 My Role: Sole Developer

I was the sole developer responsible for the entire project lifecycle:

  • Full‑Stack Development: React frontend + Firebase backend (Firestore, Auth, Messaging)
  • Map Integration: Leaflet + OpenStreetMap with terminal markers and GPS location
  • Real‑Time Seats: Synced seat availability using Firestore transactions
  • QR Code Tickets: Generated via free API and displayed for scanning
  • Driver Scanner: Camera‑based QR validation using @yudiel/react-qr-scanner
  • Push Notifications: (Optional) Firebase Cloud Messaging for real‑time alerts

Development Process

1

Requirements & Planning

Researched kombi travel pain points in Eswatini – no digital booking, uncertainty about seat availability. Planned core features: map‑based terminal selection, real‑time seat map, QR tickets, and driver validation.

2

Firestore Data Modelling

Designed collections: terminals (geo point, name), vehicles (with sub‑collection seats for real‑time status), bookings, tickets. Used composite indexes for efficient queries.

3

Map & Terminal Selection

Integrated Leaflet map, added draggable markers for Eswatini terminals (Mbabane, Manzini, Siteki, Nhlangano). Implemented click‑to‑select origin/destination with visual feedback.

4

Real‑Time Seat Selector

Built interactive seat grid that listens to Firestore changes. Used transactions to lock seats during booking, preventing double bookings.

5

QR Ticket & Scanner

Generated QR codes using public API, embedded in confirmation page. Built driver scanner that reads ticket ID and updates Firestore to mark ticket as used.

6

Testing & Deployment

Tested full flow on multiple devices, deployed to Netlify with environment variables. Wrote seed scripts to populate terminals and a demo kombi.

Key Features

Interactive Map

Choose departure and destination terminals using Leaflet + OpenStreetMap. Map centres on user’s GPS location, with fallback to Eswatini.

Terminal Markers

Four main Eswatini terminals (Mbabane, Manzini, Siteki, Nhlangano) with draggable markers for easy selection.

Real‑Time Seat Selector

Live seat availability synced with Firestore – seats become unavailable the moment they are booked.

QR Code Tickets

Each booking generates a unique QR code (via free API) that passengers can show to the driver.

Driver Scanner

Drivers can scan QR codes to instantly validate tickets and update booking status to used.

Push Notifications (Optional)

Firebase Cloud Messaging integration for real‑time alerts about booking confirmations or driver updates.

Technical Implementation

Firestore Data Model

  • terminals: { name, location: GeoPoint }
  • vehicles: { name, totalSeats, routeId? } with sub‑collection seats ({ number, status })
  • bookings: { userId, vehicleId, seatNumber, status, createdAt }
  • tickets: { bookingId, qrData, used }

Security (Firestore Rules)

During development: open rules. For production, rules are locked to authenticated users:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}

Real‑Time Seat Booking

Used Firestore transactions to prevent race conditions:

await runTransaction(db, async (transaction) => {
  const seatRef = doc(db, 'vehicles', kombiId, 'seats', seatNumber);
  const seatDoc = await transaction.get(seatRef);
  if (seatDoc.data().status !== 'available') throw 'Seat taken';
  transaction.update(seatRef, { status: 'booked', bookingId });
  transaction.set(bookingRef, bookingData);
});
